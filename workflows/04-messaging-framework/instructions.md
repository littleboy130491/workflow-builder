# 04 Messaging Framework

## Brief Description

Build the reusable messaging system that will govern headlines, proof claims, tone, value articulation, and calls to action across the site.

## Expected Input

- `outputs/02-audience-positioning/audiences.md`
- `outputs/02-audience-positioning/positioning.md`
- `outputs/03-content-direction/recommended-direction.md`
- `outputs/03-content-direction/asset-gap-list.md`

## Expected Outputs

Create the following files inside `outputs/04-messaging-framework/`:

- `messaging-framework.md`: Brand promise, value pillars, proof strategy, CTA hierarchy, and tone guidance
- `message-house.md`: Core message, support messages, proof points, and audience relevance
- `claim-validation.md`: Key claims mapped to available proof or flagged evidence gaps

## Success Criteria

- The framework gives copywriting a stable strategic baseline
- Each important claim has supporting proof or an explicit gap
- Tone and CTA logic are clear enough to keep page copy consistent

## Constraints

- Avoid inflated brand language without evidence
- Keep the messaging system practical and reusable

## Execution Notes

- Make the framework easy to apply page by page
- Distinguish between mandatory messages and optional supporting language
