# Step 11: Improvement Plan — Inputs

## Upstream Dependencies

- `outputs/09-page-analysis/summary.md` — Analysis of existing site
- `outputs/10-reference-analysis/` — Best practices from references
- `outputs/09-page-analysis/` — Individual page analyses

## Required Inputs

- Consolidated analysis of existing site weaknesses/gaps/strengths
- Reference best practices by perspective

## Output Location

- `outputs/11-improvement-plan/improvement-plan.md` — Comprehensive improvement plan
- `outputs/11-improvement-plan/priority-matrix.md` — Priority ranking of all recommendations

## Plan Structure

Each recommendation should include:
- **Category**: Design / Content / UX / Technical
- **Current Issue**: What's wrong now
- **Recommended Change**: Specific improvement
- **Reference Example**: How references handle this
- **Priority**: P0 (Critical) / P1 (Important) / P2 (Enhancement)
- **Effort**: Low / Medium / High
- **Expected Impact**: High / Medium / Low
