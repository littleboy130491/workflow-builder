# Step 11: Improvement Plan — Instructions

## Objective

Create a comprehensive, actionable improvement plan grounded in existing site analysis and reference best practices.

## Process

1. Read consolidated analysis:
   - `outputs/09-page-analysis/summary.md` for existing site issues
   - `outputs/10-reference-analysis/summary.md` for reference insights
   - Individual page analyses in `outputs/09-page-analysis/`

2. Cross-reference each weakness/gap with reference examples:
   - Identify how references solve similar problems
   - Note specific techniques or patterns to adopt

3. Develop recommendations organized by category:

   a. **Design Improvements**:
      - Layout changes
      - Typography system
      - Color palette
      - Imagery and iconography
      - Visual effects and animations

   b. **Content Improvements**:
      - Headline and messaging
      - Value proposition
      - Copy and tone
      - Content structure
      - Call-to-action phrasing

   c. **UX Improvements**:
      - Navigation structure
      - User flows
      - CTA design and placement
      - Form usability
      - Error handling

   d. **Technical Improvements**:
      - Performance optimization
      - Accessibility enhancements
      - SEO improvements
      - Mobile responsiveness
      - Code quality

4. Prioritize recommendations using impact/effort matrix:
   - **P0**: High impact, Low/Medium effort — Do first
   - **P1**: High impact, High effort — Plan carefully
   - **P2**: Medium/Low impact — Consider later

5. Document in `outputs/11-improvement-plan/improvement-plan.md`:
   - Executive summary of key changes
   - Detailed recommendations per category
   - Each recommendation includes reference examples

6. Create `outputs/11-improvement-plan/priority-matrix.md`:
   - All recommendations ranked by priority
   - Quick reference for implementation order

## Notes

- Ground every recommendation in both the analysis AND reference examples
- Be specific — "improve typography" is not actionable; "use 18px base font with 1.6 line-height" is
- Consider dependencies — some improvements enable others
- Balance quick wins with long-term investments
