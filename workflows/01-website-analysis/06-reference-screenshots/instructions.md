# Step 06: Screenshot Capture (References) — Instructions

## Objective

Capture screenshots from reference sites to understand best practices visually.

## Process

1. Read references from `outputs/04-reference-sites/references.md`

2. For each reference site:
   - Navigate to the URL
   - Capture relevant pages/sections based on its designated perspective:
     - **Design references**: Homepage hero, about page, portfolio, general design elements
     - **Content references**: Homepage, about page, service/product pages with strong copy
     - **User Flow references**: Homepage, pricing, contact, checkout flows, CTAs
     - **Technical references**: Performance-focused pages, mobile views, accessible implementations
   - Save to appropriate subfolder in `outputs/06-reference-screenshots/`

3. For each screenshot:
   - Include page/section name in filename
   - Focus on the specific aspect that makes it a good reference

4. Capture 2-5 relevant screenshots per reference site

## Notes

- Quality over quantity — capture the most instructive sections
- If a reference site has multiple good sections, capture several
- Pay attention to mobile views for technical references
- Capture before-and-after or comparison views if reference shows good patterns
