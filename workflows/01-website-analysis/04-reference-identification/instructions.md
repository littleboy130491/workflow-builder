# Step 04: Reference Identification — Instructions

## Objective

Identify reference websites for design, content, user flow, and technical best practices.

## Process

1. Check `outputs/01-url-intake/reference-urls.txt`

2. **If user provided references:**
   - Validate each URL is accessible
   - Categorize by perspective (design/content/user-flow/technical)
   - Document in `outputs/04-reference-sites/references.md`

3. **If no user references provided:**
   - Research best-in-class websites in the same industry/niche
   - Identify 2-3 references per perspective:
     - **Design**: Sites known for excellent visual design, typography, color usage
     - **Content**: Sites with exceptional copy, messaging clarity, content structure
     - **User Flow**: Sites with strong conversion patterns, intuitive navigation, clear CTAs
     - **Technical**: Sites with fast load times, excellent accessibility, clean code
   - Document in `outputs/04-reference-sites/references.md`

4. For each reference, include:
   - URL
   - Perspective category
   - Specific aspect to learn from (e.g., "hero section layout", "pricing page copy")
   - Brief justification for why it's a strong reference

## Notes

- Prefer recent, actively maintained sites
- Consider both direct competitors and aspirational examples from other industries
- Quality over quantity — one excellent reference is better than five mediocre ones
