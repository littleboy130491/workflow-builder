# Step 10: Reference Analysis — Instructions

## Objective

Analyze reference sites to extract best practices organized by perspective.

## Process

1. Read reference designations from `outputs/04-reference-sites/references.md`

2. For each perspective, analyze the reference sites:

   a. **Design References**:
      - Layout patterns and visual hierarchy
      - Typography systems (font choices, sizes, weights, spacing)
      - Color palettes and usage
      - Imagery and iconography style
      - Spacing and rhythm
      - Visual effects (shadows, gradients, animations)

   b. **Content References**:
      - Headline and subheadline strategies
      - Value proposition articulation
      - Copy tone and voice
      - Content structure and scannability
      - Social proof placement
      - Call-to-action phrasing

   c. **User Flow References**:
      - Navigation structure and labeling
      - CTA placement and design
      - Form design patterns
      - Conversion funnel optimization
      - User journey clarity
      - Error handling and feedback

   d. **Technical References**:
      - Page performance indicators
      - Accessibility implementations
      - Mobile responsiveness
      - SEO structure
      - Code organization patterns

3. Document findings per perspective in `outputs/10-reference-analysis/{perspective}.md`

4. Create `outputs/10-reference-analysis/summary.md`:
   - Key patterns across all perspectives
   - Most impactful techniques to adopt
   - Recommended approach based on references

## Notes

- Focus on patterns that can be applied to the redesign
- Note specific implementation details (CSS properties, HTML patterns)
- Identify what makes each reference effective — not just what they do, but why it works
