# Step 09: Page Analysis (Existing Site) — Instructions

## Objective

Analyze existing core pages to identify strengths, weaknesses, and gaps.

## Process

1. For each core page (read from `outputs/03-core-pages/core-pages.md`):

   a. **Visual Analysis** (from screenshot):
      - Overall layout and visual hierarchy
      - Typography choices and readability
      - Color usage and brand consistency
      - Imagery quality and relevance
      - White space usage
      - Navigation clarity

   b. **Structural Analysis** (from HTML):
      - Semantic HTML structure
      - SEO elements (title, meta, headings hierarchy)
      - Accessibility attributes
      - Code quality and organization

   c. **Content Analysis**:
      - Messaging clarity
      - Call-to-action effectiveness
      - Content gaps
      - Tone and voice consistency

   d. **UX Analysis**:
      - User flow effectiveness
      - Form usability (if applicable)
      - Loading and performance perception
      - Mobile responsiveness (if inferable)

2. Document findings per page in `outputs/09-page-analysis/{page-name}.md`

3. Create consolidated `outputs/09-page-analysis/summary.md`:
   - Top 3-5 strengths across all pages
   - Top 3-5 weaknesses across all pages
   - Top 3-5 gaps across all pages
   - Cross-page patterns observed

## Notes

- Be specific and actionable — avoid vague observations
- Reference the actual screenshots/HTML when noting issues
- Prioritize findings by impact on business goals
- Consider accessibility as critical (WCAG guidelines)
